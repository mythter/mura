Ветки

СОЗДАНИЕ

Создать ветку

git branch <branch-name>

Создать ветку и переключиться на неё

обычная команда

git checkout -b <branch-name>

новая команда

git switch -c <branch-name>

Если есть удалённая ветка, но вам нужно создать локальную и привязаться к удаленной

git branch --track <local-branch> origin/<remote-branch>

git branch <local-branch> origin/<remote-branch>

git checkout -b <local-branch> origin/<remote-branch>

Если локальная ветка с таким именем уже существует, то она будет отслеживать удалённую

git branch -f <local-branch> origin/<remote-branch>

Если есть локальная ветка и вам нужно создать удалённую с отслеживанием

git push -u origin <branch-name>

Если есть как локальная, так и удалённая ветки и вам необходимо связать их

git branch -u origin/<remote-branch-name> <local-branch-name>

Если вам нужно удалить отслеживание

git branch --unset-upstream <branch-name>

ПЕРЕМЕЩЕНИЕ

Переместить указатель ветки на другой коммит (destination-commit может быть названием ветки)

git reset --hard <destination-commit>

git branch –f <branch-name> <destination-commit>

УДАЛЕНИЕ

УДАЛИТЬ удалённую ветку (remote-name обычно origin)

git push -d <remote-name> <branch-name>

Не забудьте выполнить git fetch --all --prune или git fetch --all -p на других машинах после удаления удалённой ветки на сервере. После удаления локальной ветки с помощью git branch -d и удаления удалённой ветки с помощью git push -d origin на других машинах могут остаться "устаревшие отслеживаемые ветки" (чтобы увидеть их, выполните git branch -a). Чтобы избавиться от них, выполните git fetch --all --prune

УДАЛИТЬ локальную ветку

git branch -d <branch-name>

Опция -D это алиас для --delete --force, который удаляет ветку «независимо от её объединенного статуса (merged status)»

git branch -D <branch-name>

УДАЛИТЬ локальную удалённую ветку

git branch -dr <remote>/<branch>

длинная команда

git branch --delete --remotes <remote>/<branch>

ПЕРЕИМЕНОВАНИЕ

Переименовать текущую ветку

git branch -m <branch_name>

Переименовать другую ветки

git branch -m <old_branch_name> <new_branch_name>

Переименовать удалённую ветку

Отправить локальную ветку на удалённый репозиторий и обновить отслеживаемую ветку

git push origin -u <new_branch_name>

Удалить удалённую ветку

git push origin --delete <old_branch_name>

ПРОСМОТР

Просмотреть ветки, которые были объединены с текущей веткой

git branch --merged

Просмотреть удалённые ветки, которые были объединены с текущей веткой

git branch -r --merged

Просмотреть ветки, которые не были объединены с текущей веткой

git branch --no-merged

УДАЛЕНИЕ УСТАРЕВШИХ ВЕТОК

git remote prune origin

короткая команда

git fetch -p

длинная команда

git fetch --prune

Изменения

ИНДЕКСИРОВАНИЕ

Индексировать все изменения

git add .

Индексировать все изменения в интерактивном режиме

git add -i

Индексировать все изменения в патч режиме

git add -p

УДАЛЕНИЕ ИЗ ИНДЕКСА

Удалить изменения из Индекса (ПЕРЕМЕСТИТЬ из Индекса в Рабочую директорию)

новая команда

git restore --staged <file>

старая команда

git reset <file>

git reset HEAD <file>

Удалить изменения в Индексе

новая команда

git restore --staged .

обычная команда

git reset

старая команда

git reset HEAD

УДАЛЕНИЕ

Удалить изменения в Рабочей директории

новая команда

git restore <file>

старая команда

git checkout -- <file>

git checkout <file>

Удаление всех изменений из Рабочей директории

новая команда

git restore .

старая команда

git checkout .

Удаление изменений из Индекса и Рабочей директории

git reset --hard

git reset --hard HEAD

УДАЛИТЬ ИЗ GIT

Удалить файл из отслеживания Git, но оставить на диске

git rm --cached <file_name>

Удалить файл из репозитория Git и с диска

git rm <file_name>

Коммиты

Создать пустой коммит

git commit --allow-empty -m "пустой коммит"

Отменить последний коммит и переместить изменения в Рабочую директорию (изменения в Индексе также будут перенесены в Рабочую директорию) (по умолчанию --mixed)

git reset HEAD^

Отменить последний коммит и переместить изменения в Индексе

git reset --soft HEAD^

Отменить последний коммит и переместить изменения в Рабочую директорию

git reset --mixed HEAD^

Отменить последний коммит и УДАЛИТЬ изменения (не закоммиченные тоже)

git reset --hard HEAD^

CHERRY PICK

Копировать один коммит в текущую ветку

git cherry-pick <SHA>

Копировать диапазон коммитов в текущую ветку

git cherry-pick <SHA>..<SHA>

git config --global --list

Конфигурация имени и email

SSH

Создайте SSH ключ

-t указывает тип ключа

-b указывает размер ключа (в битах)

-C для комментария, который поможет идентифицировать ваш ключ SSH (необязательно)

-f для имени файла, в котором сохранится SSH ключ

ssh-keygen -t rsa -b 4096 -C "<ваш email>" -f $env:USERPROFILE/.ssh/<key_name>

Добавьте SSH ключ в SSH агент

Добавьте приватный SSH ключ в SSH агент

ssh-add $env:USERPROFILE/.ssh/<key_name>

Если предыдущая команда не сработала попробуйте следующее или попробуйте решение по ссылке link

Проверьте запущен ли SSH Agent

Get-Service ssh-agent

Остановите сервис SSH Agent

Stop-Service ssh-agent

Запустите SSH Agent

Start-Service ssh-agent

Добавьте публичный SSH ключ в GitHub аккаунт

  • Скопируйте содержимое файла <key_name>.pub
  • Перейдите в Settings > SSH and GPG keys > New SSH Key
  • Вставьте скопированный публичный ключ и дайте ему название по вашему выбору

Создайте файл конфигурации SSH и добавьте Host Entries

Если файла конфигурации не существует, создайте его ($env:USERPROFILE/.ssh/config)

Host github.com-<username>
  HostName github.com
  User git
  IdentityFile C:/Users/<user name>/.ssh/<key_name>
  IdentitiesOnly yes

Host github.com-<username>

  • Это имя хоста, к которому применяются эти настройки.
  • Каждый раз при подключении к этому хосту через SSH будет применяться следующая конфигурация.
  • Вы можете использовать другие настройки для других хостов (например, для других серверов).

HostName github.com

  • Указывает фактическое имя хоста для подключения.

User git

  • GitHub требует, чтобы для всех SSH-подключений в качестве пользователя было указано git.

IdentityFile C:/Users/<user name>/.ssh/<key_name>

  • Указывает путь к вашему приватному SSH-ключу, который будет использоваться для аутентификации на GitHub.

IdentitiesOnly yes

  • Заставляет SSH использовать только ключи, указанные в файле конфигурации (через IdentityFile), и игнорировать любые другие ключи, которые могут быть загружены в агент SSH.
  • Это полезно, если у вас несколько ключей и вы хотите использовать определенный ключ для определенного хоста.

Клонирование существующего репозитория или добавление удаленного к локальному

Клонировать существующий репозиторий

git clone git@github.com-<username>:<username>/<repo_name>.git

Добавить удаленный репозиторий в локальный

git remote add origin git@github.com-<username>:<username>/<repo_name>.git

Изменить ссылку удалённого репозитория

git remote set-url origin git@github.com-<username>:<username>/<repo_name>.git

И наконец то

Не забудьте указать user.name и user.email в каждом недавно клонированном или существующем ранее репозитории.

git config user.name "Ваше имя"

git config user.email "your@email.com"

Если вы настроили глобальное подписывание GPG, вы можете удалить его для локального репозитория

git config commit.gpgSign false

Если у вас есть новый GPG ключ, вы можете установить его для локального репозитория

git config user.signingkey <new-gpg-key-id>

SQL

DML (Data Manipulation Language)

Команды DML используются для хранения, извлечения, изменения и удаления данных.

SELECT - извлечение данных из базы данных

INSERT - вставить данные в базу данных

UPDATE - обновление существующих данных в базе данных

DELETE - удаление записи из базы данных

DDL (Data Definition Language)

Команды DDL используются для создания, изменения и удаления структуры объектов базы данных.

CREATE - создание объектов базы данных, таких как таблицы и представления

ALTER - изменение структуры существующей базы данных

DROP - удаление объектов из базы данных

TRUNCATE - удаление всех строк из базы данных

DCL (Data Control Language)

Команды DCL используются для реализации контроля доступа к объектам базы данных.

GRANT - предоставление пользователю права доступа к объектам базы данных

REVOKE - отзыв привилегий пользователя, которые были ранее предоставлены с помощью GRANT

Ctrl+Shift+L – удалить строку, не добавляя её в буфер обмена
Alt+<Up/Down arrows> – переместить строку вверх/вниз

Ctrl+K,F – отформатировать выделенный блок
Ctrl+K,D – отформатировать весь файл
Ctrl+K,E – очистка кода в файле (по заданным вами правилам, например, удаление ненужных using)

Ctrl+K,S – вставка сниппета (блок if, try, for и т.п.), либо заключить выделенный код в блок
Alt+Shift+<стрелки> (Alt+<выделение мышью>) - редактирование по колонкам
Ctrl+Alt+<поместить курсор мышью> - многокурсорный ввод (редактирование в нескольких местах одновременно)

Навигация

Ctrl+G – перейти к линии
Ctrl+K,K – оставить закладку
Ctrl+K,N – перейти к следующей закладке

Ctrl+, (Ctrl+T) - поиск по коду в проекте
Ctrl+Q – поиск по инструментам IDE

F12 (Ctrl+<щелчок мыши>) – перейти к определению
Ctrl+F12 – перейти к реализации
Alt+F12 – подсказка реализации во всплывающем окне
Shift+F12 – найти использования

Ctrl+Shift+пробел – подсказка по аргументам метода
Ctrl+[,S – отметить текущий файл в обозревателе решения
Ctrl+; - поиск в обозревателе решения

Ctrl+M,O - свернуть все методы
Ctrl+M,P - развернуть все методы
Ctrl+M,M – свернуть/развернуть текущий метод
Ctrl+M,L – свернуть/развернуть все блоки кода

Shift+Alt+. – Добавить следующий совпадающий текст как выделение Shift+Alt+; – Добавить весь совпадающий текст как выделение Shift+Alt+, – Удалить последнее выбранное вхождение Shift+Alt+/ – Пропустить следующее совпадающее вхождение

Для ASP.NET
Ctrl+M,G – перейти из метода контроллера в представление или наоборот

Рефакторинг

Ctrl+. (Alt+Enter) – подсказки рефакторинга
Ctrl+R,R – переименовать
Ctrl+R,M – извлечь выделенный код в метод
Ctrl+R,I - извлечь интерфейс из класса
Ctrl+R,V (Ctrl+R,O) - реорганизовать параметры метода

Сниппеты

"prop",Tab,Tab – создать свойство
"ctor",Tab,Tab – создать конструктор
Также Tab,Tab работает для других популярных ключевых слов (if, for, switch и т.п.)

Отладка

F9 – создать точку останова
Ctrl+Alt+B – окно со всеми точками останова
Ctrl+Shift+F9 – удалить все точки останова

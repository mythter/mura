Ветки

СОЗДАНИЕ

Создать ветку

git branch <branch-name>

Создать ветку и переключиться на неё

обычная команда

git checkout -b <branch-name>

новая команда

git switch -c <branch-name>

Если есть удалённая ветка, но вам нужно создать локальную и привязаться к удаленной

git branch --track <local-branch> origin/<remote-branch>

git branch <local-branch> origin/<remote-branch>

git checkout -b <local-branch> origin/<remote-branch>

Если локальная ветка с таким именем уже существует, то она будет отслеживать удалённую

git branch -f <local-branch> origin/<remote-branch>

Если есть локальная ветка и вам нужно создать удалённую с отслеживанием

git push -u origin <branch-name>

Если есть как локальная, так и удалённая ветки и вам необходимо связать их

git branch -u origin/<remote-branch-name> <local-branch-name>

Если вам нужно удалить отслеживание

git branch --unset-upstream <branch-name>

ПЕРЕМЕЩЕНИЕ

Переместить указатель ветки на другой коммит (destination-commit может быть названием ветки)

git reset --hard <destination-commit>

git branch –f <branch-name> <destination-commit>

УДАЛЕНИЕ

УДАЛИТЬ удалённую ветку (remote-name обычно origin)

git push -d <remote-name> <branch-name>

Не забудьте выполнить git fetch --all --prune или git fetch --all -p на других машинах после удаления удалённой ветки на сервере. После удаления локальной ветки с помощью git branch -d и удаления удалённой ветки с помощью git push -d origin на других машинах могут остаться "устаревшие отслеживаемые ветки" (чтобы увидеть их, выполните git branch -a). Чтобы избавиться от них, выполните git fetch --all --prune

УДАЛИТЬ локальную ветку

git branch -d <branch-name>

Опция -D это алиас для --delete --force, который удаляет ветку «независимо от её объединенного статуса (merged status)»

git branch -D <branch-name>

УДАЛИТЬ локальную удалённую ветку

git branch -dr <remote>/<branch>

длинная команда

git branch --delete --remotes <remote>/<branch>

ПЕРЕИМЕНОВАНИЕ

Переименовать текущую ветку

git branch -m <branch_name>

Переименовать другую ветки

git branch -m  <oldbranch_name> <new_branch_name>

ПРОСМОТР

Просмотреть объединённые ветки

git branch --merged

Просмотреть объединённые удалённые ветки

git branch -r --merged

Просмотреть не объединённые ветки

git branch --no-merged

УДАЛЕНИЕ УСТАРЕВШИХ ВЕТОК

git remote prune origin

короткая команда

git fetch -p

длинная команда

git fetch --prune

Изменения

ИНДЕКСИРОВАНИЕ

Индексировать все изменения

git add .

Индексировать все изменения в интерактивном режиме

git add -i

Индексировать все изменения в патч режиме

git add -p

УДАЛЕНИЕ ИЗ ИНДЕКСА

Удалить изменения из Индекса (ПЕРЕМЕСТИТЬ из Индекса в Рабочую директорию)

новая команда

git restore --staged <file>

старая команда

git reset <file>

git reset HEAD <file>

Удалить изменения в Индексе

новая команда

git restore --staged .

обычная команда

git reset

старая команда

git reset HEAD

УДАЛЕНИЕ

Удалить изменения в Рабочей директории

новая команда

git restore <file>

старая команда

git checkout -- <file>

git checkout <file>

Удаление всех изменений из Рабочей директории

новая команда

git restore .

старая команда

git checkout .

Удаление изменений из Индекса и Рабочей директории

git reset --hard

git reset --hard HEAD

Коммиты

Отменить последний коммит и переместить изменения в Рабочую директорию (изменения в Индексе также будут перенесены в Рабочую директорию) (по умолчанию --mixed)

git reset HEAD^

Отменить последний коммит и переместить изменения в Индексе

git reset --soft HEAD^

Отменить последний коммит и переместить изменения в Рабочую директорию

git reset --mixed HEAD^

Отменить последний коммит и УДАЛИТЬ изменения (не закоммиченные тоже)

git reset --hard HEAD^

CHERRY PICK

Копировать один коммит в текущую ветку

git cherry-pick <SHA>

Копировать диапазон коммитов в текущую ветку

git cherry-pick <SHA>..<SHA>

Конфигурация

Открыть глобальную конфигурацию для редактирования

git config --global --edit

Просмотреть файл глобальной конфигурации в терминале

git config --global --list

Конфигурация имени и email

git config --global user.name "Ваше Имя"

git config --global user.email test@email.com

Окончания строк

Команда git config core.autocrlf используется для изменения того, как Git обрабатывает окончания строк. Она принимает один аргумент.

git config --global core.autocrlf input|true|false

autocrlfcommitcheckoutКогда использовать
inputCRLF → LF-Unix, MacOS
trueCRLF → LFLF → CRLFWindows
false--Единственная платформа

Лог

Фильтр

По номеру

git log -<number>

По дате

git log --since=yyyy-mm-dd

git log --until="3 days ago"

git log --after=2.weeks --before=3.days

По автору

git log --author="Имя"

По паттерну сообщения коммита

git log --grep="паттерн"

По диапазону коммитов

git log <SHA>..<SHA>

По имени файла (показываются только коммиты, в которых изменялся файл)

git log <file_name>

Удалённые репозиотрии

Добавить удалённый репозиторий

git remote add origin <repository_url>

Изменить удалённый репозиторий

git remote set-url origin <repository_url>

Просмотреть удалённый репозиторий

Добавить имя удалённого репозитория

git remote

Просмотреть больше информации об удалённом репозитории

git remote -v

Удалить удалённый репозиторий

git remote rm origin

Припрятывание

СОХРАНЕНИЕ

новая команда

git stash

git stash push

git stash push -m "сообщение"

старая команда

git stash save "сообщение"

ПРОСМОТР

git stash list

ИЗВЛЕЧЕНИЕ

Извлечь содержимое из тайника (и удалить из него)

git stash pop

Извлечь содержимое из тайника (и оставить его там же)

git stash apply

УДАЛЕНИЕ

git stash drop

Теги

СОЗДАНИЕ

Создать облегчённый тег

git tag <tag_name> <SHA>

Создать тег с аннотацией

git tag -am "сообщение" <tag_name> <SHA>

Создать удалённый тег, когда существует локальный

git push origin <tag_name>

Отправить все локальные теги на удалённый репозиотрий

git push origin --tags

ПРОСМОТР

Просмотреть информацию о теге

git show <tag_name>

Просмотреть список тегов

git tag -l

Просмотреть список тегов по паттерну

git tag -l "паттерн"

Просмотреть список тегов с первой строкой каждой аннотации

git tag -n

УДАЛЕНИЕ

Удалить локальный тег

git tag -d <tag_name>

Удалить удалённый тег

git tag -d origin <tag_name>

Рабочие деревья

СОЗДАНИЕ

Создать рабочее дерево для существующей ветки используя такое имя рабочей директории

git worktree add path/to/folder/<existing-branch-name>

Создать рабочее дерево для существующей ветки используя имя отличное от рабочей директории

git worktree add path/to/folder/ <existing-branch-name>

Создать рабочее дерево с новой веткой используя такое имя рабочей директории

git worktree add -b <new-branch-name> path/to/folder/<new-branch-name>

Создать рабочее дерево с новой веткой используя имя отличное от рабочей директории

git worktree add -b <new-branch-name> path/to/folder/

ПРОСМОТР

Показать все рабочие деревья

git worktree list

УДАЛЕНИЕ

Удалить рабочее дерево

git worktree remove path/to/folder/

или (ссылка)

rm -rf path/to/worktree
git worktree prune
git branch -D <branch-name>

IDE

Visual Studio

Основные горячие клавиши Visual Studio

Редактирование

Ctrl+K,C – закомментировать строку
Ctrl+K,U – раскомментировать строку
Ctrl+D – продублировать строку
Ctrl+Shift+L – удалить строку, не добавляя её в буфер обмена
Alt+<Up/Down arrows> – переместить строку вверх/вниз

Ctrl+K,F – отформатировать выделенный блок
Ctrl+K,D – отформатировать весь файл
Ctrl+K,E – очистка кода в файле (по заданным вами правилам, например, удаление ненужных using)

Ctrl+K,S – вставка сниппета (блок if, try, for и т.п.), либо заключить выделенный код в блок
Alt+Shift+<стрелки> (Alt+<выделение мышью>) - редактирование по колонкам
Ctrl+Alt+<поместить курсор мышью> - многокурсорный ввод (редактирование в нескольких местах одновременно)

Навигация

Ctrl+G – перейти к линии
Ctrl+K,K – оставить закладку
Ctrl+K,N – перейти к следующей закладке

Ctrl+, (Ctrl+T) - поиск по коду в проекте
Ctrl+Q – поиск по инструментам IDE

F12 (Ctrl+<щелчок мыши>) – перейти к определению
Ctrl+F12 – перейти к реализации
Alt+F12 – подсказка реализации во всплывающем окне
Shift+F12 – найти использования

Ctrl+Shift+пробел – подсказка по аргументам метода
Ctrl+[,X (Ctrl+Alt+L) – отметить текущий файл в обозревателе решения
Ctrl+; - поиск в обозревателе решения

Ctrl+M,O - свернуть все методы
Ctrl+M,P - развернуть все методы
Ctrl+M,M – свернуть/развернуть текущий метод
Ctrl+M,L – свернуть/развернуть все блоки кода

Shift+Alt+. – Добавить следующий совпадающий текст как выделение Shift+Alt+; – Добавить весь совпадающий текст как выделение Shift+Alt+, – Удалить последнее выбранное вхождение Shift+Alt+/ – Пропустить следующее совпадающее вхождение

Для ASP.NET
Ctrl+M,G – перейти из метода контроллера в представление или наоборот

Рефакторинг

Ctrl+. (Alt+Enter) – подсказки рефакторинга
Ctrl+R,R – переименовать
Ctrl+R,M – извлечь выделенный код в метод
Ctrl+R,I - извлечь интерфейс из класса
Ctrl+R,V (Ctrl+R,O) - реорганизовать параметры метода

Сниппеты

"prop",Tab,Tab – создать свойство
"ctor",Tab,Tab – создать конструктор
Также Tab,Tab работает для других популярных ключевых слов (if, for, switch и т.п.)

Отладка

F9 – создать точку останова
Ctrl+Alt+B – окно со всеми точками останова
Ctrl+Shift+F9 – удалить все точки останова

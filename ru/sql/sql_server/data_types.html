Типы данных

Строковые типы данных

Тип данныхОписаниеМаксимальная длина символаХранилище
char(n)Символьные не Unicode данные фиксированной длины (n от 1 до 8000)8,000n байтов (используется один байт для каждого символа)
varchar(n)Символьные не Unicode данные переменной длины (n от 1 до 8000)8,000n байт + 2 байта
varchar(max)Символьные не Unicode данные переменной длиныдо 2 GB
nchar(n)Символьные Unicode данные фиксированной длины (n от 1 до 4000)4,0002 * n байт (используется два байта для каждого символа)
nvarchar(n)Символьные Unicode данные переменной длины (n от 1 до 4000)4,0002 * n байт + 2 байта (используется два байта для каждого символа)
nvarchar(max)Символьные Unicode данные переменной длиныдо 2 GB
binary(n)Двоичные данные фиксированной длины (n от 1 до 8000)8,000n байт
varbinary(n)Двоичные данные переменной длины (n от 1 до 8000)8,000фактическая длина введенных данных + 2 байта
varbinary(max)Двоичные данные переменной длины2GB

Числовые типы данных

Тип данныхОписаниеХранилище
bitЦелое число, может быть 0, 1 или NULL
tinyintДопускает целые числа от 0 до 2551 байт
smallintДопускает целые числа от -32 768 до 32 7672 байта
intДопускает целые числа от -2 147 483 648 до 2 147 483 6474 байта
bigintДопускает целые числа от -9 223 372 036 854 775 808 до 9 223 372 036 854 775 8078 байт
decimal(p,s)Числа с фиксированной точкой.

Допустимы числа от -10^38 +1 до 10^38 –1.

Параметр p указывает максимальное общее количество цифр, которое можно хранить (как слева, так и справа от десятичной точки). p может быть от 1 до 38. Значение по умолчанию - 18.

Параметр s указывает максимальное количество цифр, которое можно сохранить справа от десятичной точки. s может быть от 0 до p. Значение по умолчанию - 0		5-17 байт
numeric(p,s)Числа с фиксированной точкой.

Допустимы числа от -10^38 +1 до 10^38 –1.

Параметр p указывает максимальное общее количество цифр, которое можно хранить (как слева, так и справа от десятичной точки). p может быть от 1 до 38. Значение по умолчанию - 18.

Параметр s указывает максимальное количество цифр, которое можно сохранить справа от десятичной точки. s может быть от 0 до p. Значение по умолчанию - 0		5-17 байт
smallmoneyДенежные данные от -214,748.3648 до 214,748.36474 байта
moneyДенежные данные от -922,337,203,685,477.5808 до 922,337,203,685,477.58078 байт
float(n)Числа с плавающей точкой от -1,79E + 308 до 1,79E + 308.

Параметр n указывает, должно ли поле содержать 4 или 8 байтов. float(24) содержит 4-байтовое поле, а float(53) - 8-байтовое. Значение n по умолчанию - 53.		4 или 8 байт
realЧисла с плавающей точкой от -3.40E + 38 до 3.40E + 384 байта

Типы данных даты и времени

Тип данныхОписаниеХранилище
datetimeС 1 января 1753 года по 31 декабря 9999 года с точностью 3,33 мс8 байт
datetime2С 1 января 0001 года по 31 декабря 9999 года с точностью до 100 нс6-8 байт
smalldatetimeС 1 января 1900 года по 6 июня 2079 года с точностью до 1 минуты4 байта
dateХранит только дату. С 1 января 0001 г. по 31 декабря 9999 г.3 байта
timeХранит время с точностью до 100 нс3-5 байт
datetimeoffsetТо же самое, что datetime2, но со смещением часового пояса8-10 байт
timestampХранит уникальный номер, который обновляется каждый раз при создании или изменении строки. Значение timestamp основано на внутреннем времени и не соответствует реальному времени. Каждая таблица может иметь только одну переменную timestamp

Тип данныхОписание

xmlХранит данные в формате XML. Максимальный объём - 2 ГБ
cursorХранит ссылку на курсор, используемый для операций с базой данных
tableХранит набор результатов для последующей обработки
